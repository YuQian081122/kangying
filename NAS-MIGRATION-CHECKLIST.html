康鷹網站 NAS 部署清單

這版已統整可直接使用的指令，含 Git clone、初始化、首次上線、日常更新。

Repo：YuQian081122/kangying 部署：Docker Compose 日常：npm run nas:go

1) 先決條件

  • NAS 可 SSH 登入
  • 已安裝 Docker 與 Docker Compose
  • 網域 A 記錄（@ / www）指向 NAS 公網 IP
  • 路由器已轉發 80 / 443 到 NAS

2) 取得專案（可直接貼）

在 NAS 上執行：

git clone https://github.com/YuQian081122/kangying.git cd kangying

你現在可以直接 clone 這個網址，不用再猜。

3) 初始化（我已幫你建好腳本）

chmod +x scripts/*.sh npm install npm run nas:bootstrap npm run nas:doctor
  • .env.local 已建立
  • data/json、data/uploads、data/backups 已建立
  • 環境檢查通過

4) 編輯環境變數（必做）

# 編輯 .env.local ADMIN_PASSWORD=請設定強密碼或 bcrypt 雜湊 DATA_DIR=/data DATA_PATH=/volume1/docker/kangying/data

注意：DATA_PATH 一定要換成你的 NAS 實際路徑。

5) 首次上線（第一次）

npm run nas:deploy bash scripts/init-ssl.sh your-email@example.com
  • 部署成功
  • SSL 憑證申請成功
  • https://kangying.com.tw 可開啟

6) 日常維運（之後主要用）

# 日常更新（建議） npm run nas:go # 異常回滾 npm run nas:rollback # 手動備份 npm run backup # 從備份還原 npm run nas:restore -- ./data/backups/backup-YYYY-MM-DD_HHMMSS.tar.gz

平常固定跑 npm run nas:go 就好。

7) 一條龍指令（從 0 到可更新）

git clone https://github.com/YuQian081122/kangying.git cd kangying chmod +x scripts/*.sh npm install npm run nas:bootstrap npm run nas:doctor npm run nas:deploy bash scripts/init-ssl.sh your-email@example.com # 之後更新 npm run nas:go

文件版本：v3（已修正指令、穩定版型）

NAS 部署一頁清單（最精簡版）

你只要照這一頁做，就可以從 Windows 專案順利搬到 NAS 上線。

網域：kangying.com.tw 架構：Next.js + Docker + Nginx + Certbot 日常更新：npm run nas:go

1) 先決條件

  • NAS 可 SSH 登入
  • 已安裝 Docker / Docker Compose
  • DNS 已指向 NAS 公網 IP（@ 與 www）
  • 路由器轉發 80 與 443

2) 取得專案（你問的 git clone）

目前你的本機資料夾不是 git 倉庫，所以有兩種做法：

  • A. 有遠端 Git：請把網址填進下面指令。
  • B. 沒有遠端 Git：直接把整個專案資料夾上傳到 NAS。
# A. 有 Git 遠端時（把 URL 換成你的） git clone https://<your-git-url>/web.git cd web # B. 沒有 Git 遠端時 # 直接將目前這個資料夾完整上傳到 NAS，然後進入該目錄

3) 初始化（我已幫你做腳本）

在專案目錄執行：

chmod +x scripts/*.sh npm run nas:bootstrap npm run nas:doctor
  • .env.local 已建立（若不存在）
  • `data/json`, `data/uploads`, `data/backups` 已建立
  • 環境檢查通過

4) 先改這三個值

# 編輯 .env.local ADMIN_PASSWORD=請換成強密碼或bcrypt雜湊 DATA_DIR=/data DATA_PATH=/volume1/docker/kangying/data

注意： DATA_PATH 一定要改成你的 NAS 真實路徑。

  • 已設定 ADMIN_PASSWORD
  • 已設定正確 DATA_PATH

5) 首次上線指令（完整）

npm run nas:deploy bash scripts/init-ssl.sh your-email@example.com
  • 部署成功
  • SSL 申請成功
  • https://kangying.com.tw 可開啟

6) 日常維運（你之後只要記這區）

# 日常更新（建議） npm run nas:go # 出問題回滾 npm run nas:rollback # 手動備份 npm run backup # 從備份還原 npm run nas:restore -- ./data/backups/backup-YYYY-MM-DD_HHMMSS.tar.gz

建議：平常就固定跑 npm run nas:go。

7) 一鍵複製版（從 0 到更新）

# 1) 進專案目錄 cd /path/to/web # 2) 初始化 + 檢查 chmod +x scripts/*.sh npm run nas:bootstrap npm run nas:doctor # 3) 首次部署（第一次） npm run nas:deploy bash scripts/init-ssl.sh your-email@example.com # 4) 後續更新（平常） npm run nas:go

文件版本：v2（穩定不跑版版型）

Windows -> NAS 遷移清單

康鷹空間資訊網站部署流程（可勾選執行版）

目標：穩定上線 模式：Docker Compose 網域：kangying.com.tw

1) NAS 基礎準備

  • NAS 已啟用 SSH
  • NAS 已安裝 Docker / Container 功能
  • 已建立持久化資料夾（如 /volume1/docker/kangying）
  • 路由器可設定 Port Forward（80 / 443）

2) 專案搬到 NAS

  • 專案已放到 NAS（git clone / 上傳）
  • 可透過 SSH 進入專案目錄
  • 確認有 docker-compose.yml、Dockerfile、scripts/

3) 環境變數（最重要）

  • 建立 .env.local（由 .env.example 複製）
  • 設定 ADMIN_PASSWORD（建議 bcrypt）
  • 設定 DATA_PATH=/volume1/docker/kangying/data
  • 確認 DATA_DIR=/data

4) DNS 與網路

  • kangying.com.tw A 記錄指向 NAS 公網 IP
  • www A 記錄指向同一 IP
  • 路由器已轉發 80 / 443 到 NAS
  • NAS 防火牆允許 80 / 443

5) 首次部署

chmod +x scripts/*.sh npm run nas:doctor npm run nas:deploy bash scripts/init-ssl.sh 你的信箱
  • 部署命令全部成功
  • SSL 憑證申請成功

6) 上線驗證

  • https://kangying.com.tw 可開啟
  • 首頁、產品列表、產品內頁正常
  • /upload 可登入與儲存
  • 圖片與影片可正常顯示

7) 日常維運（之後主要用）

npm run nas:go npm run nas:rollback npm run nas:restore -- <backup.tar.gz>
  • 更新使用 nas:go（含備份與失敗自動回滾）
  • 異常時會使用 nas:rollback
  • 備份可用 nas:restore 驗證還原

8) 排程與演練

# 每日凌晨 3 點備份 0 3 * * * cd /path/to/web && bash scripts/backup.sh >> /var/log/kangying-backup.log 2>&1
  • 已設定每日自動備份
  • 每月完成一次還原演練
建議操作順序： 平常只要跑 npm run nas:go。如果更新失敗，先看 log，再執行 npm run nas:rollback。
